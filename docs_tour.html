Fluxo de Execucao

O programa parte da entrada lexica, gera a fita de tokens e entao executa o pipeline sintatico.

1. LexicoGera fita.csv e tabela_simbolos.csv.
2. GramaticaCarrega e normaliza a GLC.
3. SLRCalcula FIRST/FOLLOW, itens LR(0), transicoes e tabela.
4. ParserExecuta shift/reduce e anota a TS.
5. ExtrasSemantica, codigo intermediario e otimizacao.
fita.csv -> GrammarLoader -> GrammarNormalizer -> SLRTableBuilder
         -> SyntacticAnalyzer -> SemanticAnalyzer -> IntermediateCodeOptimizer

Etapa 1 - Reconhecimento Sintatico SLR

GrammarLoader.java

Carrega a GLC

Le as producoes, normaliza simbolos e trata epsilon como producao vazia.

if (!normalizedRight.contains(" ")) {
    String symbol = normalizeSymbol(normalizedRight);
    return EPSILON.equals(symbol) ? List.of() : List.of(symbol);
}

Ponto importante: epsilon nao e token da fita; ele representa lado direito vazio.

FirstFollowCalculator.java

Calcula FIRST/FOLLOW

FIRST indica inicios possiveis; FOLLOW indica o que pode aparecer apos um nao terminal.

follow.get(grammar.startSymbol()).add("$");

do {
    changed = computeFirst(grammar, first) || computeFollow(grammar, first, follow);
} while (changed);
SLRTableBuilder.java

Monta itens, transicoes e tabela SLR

Constroi estados LR(0), registra ACTION/GOTO e detecta conflitos.

if (grammar.isTerminal(nextSymbol)) {

SemanticAnalyzer.java

Caracteristica semantica

Valida se cada simbolo reconhecido tem categoria, nome, tipo e se nao ha duplicidade por categoria.

// Caracteristica semantica: cada lexema reconhecido deve ter tipo e nome unicos por categoria.
if (isBlank(entry.getSemanticName())) {
    errors.add("Linha " + entry.getLine() + ": simbolo sem nome semantico.");

String key = normalize(entry.getSyntaxCategory()) + "::" + normalize(entry.getSemanticName());

Etapa 3 - Codigo Intermediario

SyntacticAnalyzer.java

Geracao durante reducoes

A demonstracao foi aplicada na regra TOKEN -> terminal. Cada reducao gera um temporario e depois associa esse temporario ao rotulo do token. 

// Demonstracao da etapa 3 para a regra TOKEN -> terminal.

intermediateCode.add(temp + " = TOKEN(" + terminal + ", " + lexeme + ")");
intermediateCode.add(identifier + " = " + temp);

Etapa 4 - Otimizacao

IntermediateCodeOptimizer.java

Propagacao simples de copia

A estrategia remove temporarios usados somente para copiar uma expressao para outro rotulo. 

// Etapa 4: propagacao de copia remove temporarios usados uma unica vez.
if (next.endsWith(copySuffix)) {
    String target = next.substring(0, next.length() - copySuffix.length());
    optimized.add(target + " = " + expression);
    i++;
    continue;

Pontos Para Arguicao

  • A fita do sintatico vem de fita.csv e termina com $.
  • epsilon e producao vazia, por isso nao deve ser consumido como token.
  • FIRST/FOLLOW sustentam as reducoes da tabela SLR.
  • shift consome token; reduce reconhece uma estrutura sintatica.
  • A tabela de simbolos recebe categoria sintatica, observacao de reducao e atributos semanticos.
  • A analise semantica valida nome, tipo e duplicidade por categoria.
  • O codigo intermediario e gerado em uma reducao especifica para demonstracao.
  • A otimizacao elimina temporarios redundantes por propagacao de copia.
